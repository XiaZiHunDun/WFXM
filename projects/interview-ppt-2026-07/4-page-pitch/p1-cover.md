## 二、画面填充（直接拷进 PPT 模板）

### 1. 标题（48pt 居中）

```
WFXM Butler
```

### 2. 灵魂标签（最大字号 60-72pt，居中）

```
跨进程工具路由  +  多源记忆治理
```

### 3. 左栏 · 痛点（20pt）

- 远程管理多项目时，微信最方便但工具都在电脑端
- 现成方案**五不贴**：ChatGPT 没工具 / Cursor 要开电脑 / LangChain 是框架不是产品
- 工具过 10 个后单 Agent 调不动

### 4. 右栏 · Why Agent（20pt）

- **意图模糊**：自然语言需求无法预定义 if-else
- **工具组合**：单步工具已不够，需要 5+ 工具串/并联
- **跨会话状态**：长程任务要记上下文，传统脚本无记忆
- **权限分级**：多 Owner / 多项目 / 多子代理，权限边界必须工程化

### 5. 底部脚注（18pt 灰）

```
1254 Py · 自建 Loop · 0 框架依赖 · 7 runtime jobs
```

## 三、口播稿（45 秒，照念即可）

> **【0-10s · 定调】**
> "我做的是 **WFXM Butler**——自托管的多项目 AI 管家。**核心标签是'跨进程工具路由 + 多源记忆治理'**——前者解决 Agent 在多权限域里安全调度工具，后者解决 Agent 在长会话里不丢上下文。"
>
> **【10-35s · 痛点共鸣】**
> "为什么必须自建？因为三个痛点没人解决：**第一**，远程管理多项目时微信最方便，但工具都在电脑端；**第二**，现成方案五不贴——ChatGPT 没工具、Cursor 要开电脑、LangChain 是框架不是产品；**第三**，工具过 10 个后单 Agent 调不动。"
>
> **【35-45s · Why Agent 收口】**
> "为什么必须 Agent？意图模糊 + 工具组合 + 跨会话 + 权限分级——传统脚本解决不了。今天 2 分钟我讲后面 3 页。"

## 五、可能追问预案

| 追问 | 一句话回答 |
|------|-----------|
| 什么叫"跨进程"？ | Agent 在多 Owner 对话 / dev 子代理 / 外部 MCP 之间调度，每个进程有独立权限域 |
| 为什么不直接用 ChatGPT？ | ChatGPT 没有工具调用能力、没有微信入口、不能跨项目切换 |
| LangChain 不行吗？ | LangChain 是框架不是产品，单租户自托管场景下框架自带能力是负债 |
| 1254 个 Py 文件合理吗？ | 自建 Loop 涉及工具注册、权限门控、记忆治理、微信网关等 8 个子系统 |
