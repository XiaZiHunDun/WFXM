# 4 页 PPT Pitch · 浓缩版

> **用途**：当面试窗口极短（30 秒电梯 / 2 分钟简短介绍 / HR 面）时使用。
> **对比 17 页版**：砍掉 80% 内容，留 20% 最强论点；每页 30-45 秒，总时长 2-3 分钟。
> **何时用 vs 17 页版**：面试官说"3 分钟介绍下你自己" / 投递简历后第一轮电话面 / 大会 lightning talk。

## 4 页取舍逻辑（最重要）

### 我砍掉的（vs 17 页版）

| 砍掉 | 原因 |
|------|------|
| 5 个不做清单（→ P5） | 重要但不致命，省略 |
| 4 个认知颠覆（→ P16） | 只留 1 个最强的 |
| 3 个 baseline 对比（→ P13） | 只留 1 个最强的 |
| 3 个真实 bug（→ P11） | 只留 1 个最戏剧化的 |
| System Prompt 结构（→ P10） | 太细节，归入 A 的"关键代码" |
| 角色表（→ P5） | 用一句话覆盖 |
| 反思/设计不足（→ P15） | 留到 Q&A 追问时讲 |

### 我保留的（核心 4 页）

| 保留 | 原因 |
|------|------|
| **P1 封面 + 标签 + 痛点 + Why Agent** | 10 秒定调 + 痛点共鸣是面试官第一秒判断"有没有意思"的关键 |
| **P2 架构 + 双挑战** | 整个项目的"灵魂可视化"，面试官从此页决定要不要继续听 |
| **P3 量化结果 + 真实 Bug + Baseline** | 用真实数字和真实事故回答"靠谱吗" |
| **P4 认知颠覆 + 标签回锚** | 留钩子，差异化最强的收口 |

## 与 17 页版的对应关系

```
17 页版                          4 页版
─────────────────────────────────────────
P1 封面 + P2 痛点 + P3 Why       →  P1 浓缩（去掉 Why 部分细节）
P6 架构 + P7 自建 + P8 记忆 + P9 工具 → P2 浓缩（关键代码 8→4 行）
P12 量化 + P14 业务结果 + P11 Bug + P13 Baseline → P3 浓缩（3 合 1）
P16 认知 + P17 收口              →  P4 浓缩（1 个认知 + 标签回锚）
```

## 何时用哪个版本

| 场景 | 用版本 |
|------|--------|
| HR 面 / 简历投递后第一轮 / 大会 lightning | **4 页版** |
| 技术初面（30-45 分钟） | 17 页版 |
| 技术终面（1 小时+） | 17 页版 + cheat-sheet |
| 临时电梯（约到面聊） | **4 页版 cheat-sheet** 即可 |
