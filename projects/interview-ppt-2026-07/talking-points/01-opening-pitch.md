# 01 · 开场白（45-60 秒）

> **用途**：面试官说"先自我介绍一下"或"讲讲你最近的项目"时用。**实际口径 45-60 秒**，原标"30 秒"是压缩极限不可达；面试官给的开场窗口通常是 60-90 秒。
>
> **节奏建议**：前 25 秒铺垫定位 + 标签，后 30 秒讲挑战和解法，留 5 秒过渡到深讲。

---

## 主推版本（已校准，45-60 秒）

> **"我最近一年多在做 WFXM Butler，一个自托管的多项目 AI 管家**。
>
> **核心场景是：你在微信上发消息，它能理解意图、跨工具调度、跨项目切换、跨会话记住上下文**。
>
> **技术上最核心的两个挑战**——
>
> **一是'跨进程工具路由'**：当工具数超过 100、Agent 横跨 Owner 对话 / dev 子代理 / 外部 MCP，怎么保证既不越权、又不幻觉；
>
> **二是'多源记忆治理'**：记忆来自三层（Owner 画像 / 项目 MEMORY / 会话 transcript），怎么在压缩、漂移、抽取下不失真。
>
> **我的解法是'自建 Agent Loop + 文件 SSOT'**——1254 个 Py 文件，不用 LangGraph，理由是单租户自托管场景下框架自带能力是负债而非资产。
>
> **今天演示 backlog 已交付 9/10 项**，主流程跑完。接下来 2-3 分钟我展开讲第二个挑战——'多源记忆治理'。"

---

## ⚠ 30 秒开场关键要点

| 必须包含 | 可选包含 | 不要包含 |
|----------|----------|----------|
| 项目名 + 一句话定位 | 数字成果（9/10 backlog） | 详细技术细节（留给深讲） |
| 技术标签（"跨进程工具路由 + 多源记忆治理"）| 自研理由（如果不打断深讲） | 完整功能列表 |
| "接下来展开讲 XX" | demo 邀请 | 商业价值 / ROI |

## 节奏控制

```
0-5s   : 项目名 + 一句话定位
5-15s  : 两个核心挑战（标签 + 简短解释）
15-25s : 我的解法（自建 Loop + 文件 SSOT）+ 关键数字
25-30s : 收口（"接下来展开讲 XX"）
```
