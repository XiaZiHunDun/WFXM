## 阅读任务接收规则

### 任务接收判断
1. 接收主控调度任务（携带章节范围）
2. 判断任务类型：
   - **单章任务**（chXXX）→ 直接执行阅读+评论
   - **多章任务**（chXXX-chYYY）→ 启动并行执行模式

### 任务接收格式
```
主控调度 → 读者N → 阅读ch001-ch010
```

### 任务确认清单
- [ ] 确认章节范围
- [ ] 确认输出路径（意见仓库位置）
- [ ] 确认是否有P0问题优先检查要求
- [ ] 确认评论截止时间（如有）

## 并行启动同伴规则

### 启动条件
当任务范围 ≥ 3章时，必须启动同伴读者并行执行

### 启动策略
| 任务规模 | 启动同伴数 | 每人负责章节 |
|---------|-----------|------------|
| 3-5章 | 2-3人 | 1-2章/人 |
| 6-10章 | 5-8人 | 1-2章/人 |
| 11-20章 | 10-15人 | 1-2章/人 |

### 启动流程
1. 计算任务范围内的章节数 N
2. 计算可用的同伴读者数 M
3. 每人分配章节数 = ceil(N / M)
4. 使用 Agent 工具启动同伴读者 Subagent
5. 设置 run_in_background: true
6. 传递：章节内容路径、输出路径、评论格式要求

### 启动示例
```
任务：阅读ch001-ch010（共10章），可用同伴9人
执行：
  读者N（本人）→ ch001-ch002
  读者O → ch003-ch004
  读者P → ch005-ch006
  读者Q → ch007-ch008
  读者R → ch009-ch010

启动方式：
Agent(读者O, run_in_background: true, 携带ch003-ch004内容)
Agent(读者P, run_in_background: true, 携带ch005-ch006内容)
...
```

### 紧急备忘
**绝不在同一任务中串行等待，必须立即启动所有并行Agent**

## 质量评估标准（S/A/B三级）

| 等级 | 完整度 | 处理 |
|------|--------|------|
| S级 | >90% | 深度足够，剧情合理 |
| A级 | 70%-90% | 整体可接受，个别鸡汤 |
| B级 | 50%-70% | 鸡汤过多，深度不足 |
| 不合格 | <50% | 套路化严重，打回重写 |

## 评论聚合规则

### 多读者意见合并
当同一章节被多名读者评论时：
- **共鸣问题**：3人以上同一问题 → P1级问题
- **分歧问题**：读者类型不同导致的分歧 → 记录为"读者类型差异"
- **弃书指数**：取最高值（最严格标准）

### 弃书率阈值
| 弃书指数 | 处理 |
|---------|------|
| ≤5 | 合格，进入下一批 |
| 6-7 | 警告，需修改 |
| ≥8 | 不合格，打回重新创作 |
