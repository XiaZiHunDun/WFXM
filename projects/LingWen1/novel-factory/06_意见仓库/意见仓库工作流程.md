# 意见仓库工作流程

## 概述

意见仓库是小说工作室的核心反馈枢纽，接收来自审核部门、作家部门、读者部门和汇总部门的六类意见，进行分类、分发、跟踪和闭环管理。

## 意见处理流程

### 流程图

```
接收 → 分类 → 分发 → 跟踪 → 闭环
  ↓      ↓      ↓      ↓      ↓
 录入   优先级   指定   状态   归档
 意见   分级   处理人   更新   定稿
```

### 1. 接收（Receive）

- 审核员完成审核后，将意见记录写入对应目录
- 文件命名格式：`{章节标识}_{审核员}_{审核类型}.md`
- 示例：`ch001_审核员A_审核.md`

### 2. 分类（Classify）

根据意见内容进行分类：
- **按来源**：审核部门/作家部门/读者部门/汇总部门
- **按对象**：大纲类/正文类
- **按优先级**：P0/P1/P2/P3

### 3. 分发（Dispatch）

| 优先级 | 分发规则 | 处理时效 |
|--------|---------|---------|
| P0 | 立即通知处理人，Agent并行修复 | 1小时内 |
| P1 | 当批次处理，优先于P2/P3 | 4小时内 |
| P2 | 纳入下一批次修改计划 | 24小时内 |
| P3 | 记录在案，纳入优化迭代 | 72小时内 |

### 4. 跟踪（Track）

通过 `run_opinion.sh` 工具跟踪：
```bash
./run_opinion.sh pending     # 查看待处理意见
./run_opinion.sh list <类型> # 按类型查看意见清单
```

### 5. 闭环（Close）

意见闭环条件：
- **P0**：修复完成并通过复审
- **P1**：修复完成或经评估暂不处理（有记录）
- **P2/P3**：纳入下一迭代版本

## P0/P1/P2/P3 分级处理规则

### P0 - 紧急（硬伤/死锁）

**定义**：逻辑硬伤、设定冲突、人设崩塌、安全问题

**处理规则**：
- 立即暂停当前流程
- 启动多Agent并行修复
- 主控亲自跟踪
- 修复后必须复审

**示例**：
```
P0 | 逻辑矛盾 | ch051中星月性别前后不一
P0 | 人设崩塌 | ch080中林夜行为与其孤儿背景矛盾
P0 | 设定冲突 | 境界体系在ch100与ch105描述不一致
```

### P1 - 高优（影响阅读/市场）

**定义**：影响读者体验、文笔问题、节奏问题

**处理规则**：
- 当批次优先处理
- 启动对应作家修复
- 修复后抽样复审

**示例**：
```
P1 | 节奏拖沓 | ch035-ch040打斗场景过于冗长
P1 | 文笔优化 | ch060多处描写可以更精炼
P1 | 读者反馈 | ch055情感铺垫不足
```

### P2 - 中优（可改进项）

**定义**：优化建议、风格统一、伏笔强化

**处理规则**：
- 纳入下一批次修改计划
- 可与P1问题一起处理

**示例**：
```
P2 | 风格统一 | ch070-ch080对话风格需统一
P2 | 伏笔强化 | ch045伏笔可在ch090呼应
P2 | 轻微优化 | ch101行"它们还在动"改为"瞳孔还在动"
```

### P3 - 低优（记录项）

**定义**：未来优化方向、版本迭代储备

**处理规则**：
- 记录在案
- 纳入大版本迭代

**示例**：
```
P3 | 长期优化 | 建议在修订版中考虑增加配角戏份
P3 | 市场调研 | 读者反馈主角出场率偏低
```

## 意见合并与去重机制

### 合并规则

1. **同章节同问题**：多条相同意见合并为一条，标注来源数量
   ```
   合并前：ch001 审核员A → 节奏拖沓
   合并前：ch001 审核员B → 节奏拖沓
   合并后：ch001 节奏拖沓（2人反馈）
   ```

2. **跨章节同问题**：跨章节相同问题可分配给同一作家并行处理

### 去重规则

1. **精确去重**：同章节+同问题类型+同优先级 → 保留一条
2. **优先级升级**：低优先级意见发现实质问题时，升级为高优先级
3. **过期意见**：同一问题修复后，新产生的不同意见视为新意见

### 合并执行时机

- 审核完成后立即合并（自动）
- 批次修改前合并（自动）
- 主控手动触发合并（按需）

## 意见状态机

```
[新建] → [待分派] → [处理中] → [待复审] → [已闭环]
   ↓         ↓          ↓         ↓
[合并]   [已分派]    [修复完成] [重新打开]
```

### 状态说明

| 状态 | 说明 | 触发条件 |
|------|------|---------|
| 新建 | 意见刚录入 | 审核完成 |
| 待分派 | 等待分配处理人 | 录入完成 |
| 已分派 | 已指定处理人 | assign命令 |
| 处理中 | 处理人正在修复 | 开始修改 |
| 修复完成 | 修复已提交 | 作家提交 |
| 待复审 | 等待复审 | 修复完成 |
| 已闭环 | 审核通过/评估通过 | 复审通过 |
| 重新打开 | 问题未解决 | 复审不通过 |
| 合并 | 与其他意见合并 | 主控合并 |

## 调度命令

```bash
# 查看意见清单
./run_opinion.sh list <类型>   # 类型: outline/volume/stage/content/writer/reader/summary

# 查看待处理意见
./run_opinion.sh pending

# 分配意见处理
./run_opinion.sh assign <章节> <处理人>

# 标记意见已解决
./run_opinion.sh resolve <意见ID>

# 生成意见汇总报告
./run_opinion.sh report
```
