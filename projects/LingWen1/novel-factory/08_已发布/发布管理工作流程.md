# 发布管理工作流程

> **目的**：规范化已发布版本的管理，确保可追溯、可对比、可回滚

---

## 一、版本命名规范

### 格式
```
{项目名}_{版本标识}_{日期}
示例：全文终稿_星陨纪元_v1.0_20260514
```

### 版本标识

| 标识 | 含义 | 触发条件 |
|------|------|---------|
| v1.0 | 初版发布 | 首次全文定稿 |
| v2.0 | 修订版 | 重大修改后重新发布 |
| v3.0 | 三版 | 再次重大修改 |
| final | 最终版 | 经过多轮验证不再修改 |

### 版本状态定义

```
草稿 → 审核中 → 已发布 → 已归档
```

## 二、发布前检查清单

### 必检项（全部通过才能发布）

```
□ workflow_state.json 状态为 PHASE_COMPLETE
□ 所有章节已完成审核（completed列表完整）
□ 未解决意见 ≤ 5条
□ 无P0级未解决问题
□ 汇总部门已完成全文汇总
□ 汇总校验报告已生成
□ 版本编号已确定
□ 发布记录已创建
□ 完整正文合并已完成（merge命令）
```

### 合并验收（v2.0新增）

发布前必须执行正文合并，确保完整正文已生成为单文件：

```
发布流程：
1. 执行 merge 命令 → 生成完整正文文件
2. 验证合并结果（行数、章节数）
3. 确认文件大小合理（>50MB为正常）
4. 归档完整正文文件
```

**合并命令：**
```bash
./run_publish.sh merge v3.0              # 合并全部360章
./run_publish.sh merge v3.0 --range 1-120  # 只合并卷1
```

**验收标准：**
| 检查项 | 标准 | 问题级别 |
|--------|------|---------|
| 章节数量 | 360章（ch000-ch360） | P0 |
| 文件行数 | >30000行 | P1 |
| 文件大小 | >50MB（约100-150万字） | P2 |
| 章节连续性 | 无缺失章节 | P0 |

### 章节文件命名校验（v2.0新增）

**校验命令：**
```bash
./run_check_naming.sh                  # 检查命名问题
./run_publish.sh check-naming          # 同上（别名）
```

**校验内容：**
| 检查项 | 说明 | 问题级别 |
|--------|------|---------|
| 文件缺失 | ch078等文件不存在 | P0 |
| 命名错位 | 文件名与章节标题号不一致 | P1 |
| 重复章节 | 多个文件使用同一章节号 | P0 |
| 章节覆盖 | 实际覆盖章节号1-360是否完整 | P0 |

**常见问题修复：**
```bash
# 问题文件示例：
# ch169.md 标题为"第一百七十章"但文件编号是169
# 需要重命名为 ch170.md

# 修复方法：
mv ch169.md ch169_backup.md
mv ch169.md ch170.md  # 如果ch170不存在
```

**命名规范：**
- 文件名格式：`ch{章节号:3d}.md`（如 ch001.md, ch170.md）
- 章节号必须与文件内第一行标题章节号一致
- 禁止文件编号与内容章节号错位

### 发布审批流程

```
主控执行检查 → 全部通过 → 生成发布确认 → 用户确认 → 执行发布
```

## 三、发布记录格式

每个发布版本必须创建记录文件：

```markdown
# 发布记录：{项目名} v{版本}

## 基本信息
- 版本：v{版本}
- 发布日期：{YYYY-MM-DD}
- 发布人：主控Agent
- 发布原因：{首次发布/修复发布/终版发布}

## 内容概述
- 总章节数：{N}章
- 总字数：{字数}
- 状态：{草稿/已发布/已归档}

## 审核情况
- 审核批次数：{N}
- 发现问题数：P0:{N} / P1:{N} / P2:{N}
- 修复率：{百分比}

## 变更记录
- v{版本}（{日期}）：{变更说明}
- v{上一版本}（{日期}）：{上一版本说明}

## 归档信息
- 归档日期：{YYYY-MM-DD}
- 归档位置：/path/to/archive
- 归档原因：{定期归档/项目结束/版本更新}
```

## 四、归档标准

### 归档触发条件

| 条件 | 说明 |
|------|------|
| 新版本发布 | 上一版本自动归档 |
| 项目结束 | 所有版本统一归档 |
| 定期归档 | 每30天检查一次 |

### 归档操作

```
1. 执行 merge 命令生成完整正文文件
2. 复制完整正文到归档目录
3. 复制全文汇总到归档目录
4. 更新发布记录的归档信息
5. 标记原文件为归档状态（只读）
6. 更新 workflow_state.json 的归档状态
```

### 归档目录结构（v2.0更新）

```
08_已发布/
├── 归档/
│   ├── 星陨纪元_v1.0_20260514/
│   │   ├── 全文正文_星陨纪元_v1.0.md    # 完整正文（合并后）
│   │   ├── 全文终稿_星陨纪元_v1.0.md    # 全文汇总
│   │   └── 发布记录.md
│   └── 星陨纪元_v3.0_20260515/
│       ├── 全文正文_星陨纪元_v3.0.md    # 完整正文（合并后）✅
│       ├── 全文终稿_星陨纪元_v3.0.md    # 全文汇总
│       └── 发布记录.md
└── 当前版本.md（指向最新版本）
```

## 五、版本对比规则

### 对比维度

| 维度 | 内容 |
|------|------|
| 章节数 | 新增/删除/修改的章节 |
| 字数 | 整体字数变化 |
| 问题修复 | 修复的问题列表 |
| 审核意见 | 未解决意见对比 |

### 对比输出格式

```markdown
## 版本对比：v{A} vs v{B}

### 基本信息
- v{A}：{日期}，{N}章，{字数}字
- v{B}：{日期}，{N}章，{字数}字
- 差异：{+/-章节数}，{+/-字数}

### 章节变更
| 操作 | 章节 | 说明 |
|------|------|------|
| 新增 | chXXX | {说明} |
| 删除 | chXXX | {说明} |
| 修改 | chXXX | {说明} |

### 问题修复对比
| 问题 | v{A}状态 | v{B}状态 |
|------|---------|---------|
| P0逻辑矛盾 | 未修复 | 已修复 |
| P1衔接断裂 | 未修复 | 已修复 |
```

## 六、脚本命令

```bash
# 发布前检查
./run_publish.sh check v1.0

# 检查章节文件命名
./run_check_naming.sh
./run_publish.sh check-naming

# 合并正文为单文件
./run_publish.sh merge v3.0
./run_publish.sh merge v3.0 --range 1-120

# 归档指定版本
./run_publish.sh archive v1.0

# 列出所有发布版本
./run_publish.sh list

# 对比两个版本
./run_publish.sh diff v1.0 v2.0
```
