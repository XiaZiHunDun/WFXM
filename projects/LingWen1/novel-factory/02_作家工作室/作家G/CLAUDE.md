## 并行执行规则（关键）

### 接收任务时的判断
当被分配修改任务时，检查任务范围：
- **单章节**（如：修改ch001）→ 直接执行
- **多章节**（如：修改ch001-ch010）→ **必须启动多个Subagent并行执行**

**执行示例（修改10章）：**
```
主控分配：作家A→ch021, 作家B→ch022, 作家C→ch023, ...
10个作家同时启动，各自修改分配的1章
```

**禁止**：一个作家串行修改多个章节（除非只有1-2章待修改）

### Subagent启动方式
使用 `Agent` 工具启动同伴作家，参数：
- `run_in_background: true`（后台并行）
- 携带修改任务和问题清单

### 紧急备忘
**绝不在同一任务中串行修改多个章节，必须立即启动所有并行Agent**

## 反馈循环处理规则

### 接收审核反馈时
1. 读取意见仓库中的审核记录
2. 识别问题类型：
   - **逻辑问题**（如：性别矛盾、情节跳跃）→ 优先修复
   - **重复问题**（如：场景重复）→ 精简或合并
   - **缺失问题**（如：背景未交代）→ 补充铺垫
3. 修复后标注修改内容
4. 如有问题无法修复，报告原因

### 修改完成报告格式
```
## 修改完成报告

### 修改章节
- ch001：修改内容...
- ch002：修改内容...

### 问题修复状态
- 问题1（性别矛盾）：✅ 已修复
- 问题2（衔接断裂）：✅ 已修复
```

## S/A/B质量评估标准

| 等级 | 完整度 | 处理 |
|------|--------|------|
| S级 | >90% | 直接提交审核，标记高质量 |
| A级 | 70%-90% | 提交审核，附自评说明修改点 |
| B级 | 50%-70% | 修改后再自评 |
| 不合格 | <50% | 打回重写，记录返工 |

## 自评机制

作家完成章节后按S/A/B标准自评，提交审核时附带：
- 自评等级
- 自评意见（指出修改点或亮点）

审核结果与自评对比记入能力档案。
