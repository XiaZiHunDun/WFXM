# 灵感部门工作流程

## 部门架构

| 角色 | 专项 | 核心职责 |
|------|------|---------|
| 灵感A | 类型专家 | 类型/冲突/卖点/受众分析 |
| 灵感B | 幻想专家 | 世界观/力量体系/环境设定 |
| 灵感C | 结构专家 | 时间线/伏笔/叙事结构 |
| 主编 | 轮值制 | 综合整合/质量评定/版本管理 |

## 主编轮值规则

### 轮值顺序
```
项目1 → 灵感A担任主编
项目2 → 灵感B担任主编
项目3 → 灵感C担任主编
项目4 → 灵感A担任主编（循环）
```

### 主编职责
1. **综合整合**：汇总三方输出，形成立项包
2. **质量评定**：S/A/B级评定
3. **版本管理**：v1.0/v2.0/v3.0/final
4. **裁决权**：对最终立项包有裁决权

## S/A/B三级质量评估

| 等级 | 完整度 | 定义 | 处理方式 |
|------|--------|------|---------|
| **S级** | >90% | 方向正确，要素齐全 | 直接进入整合 |
| **A级** | 70%-90% | 基本完整，有小缺陷 | 主编小幅调整 |
| **B级** | 50%-70% | 缺少关键要素 | 返回重做（不计入迭代） |
| **不合格** | <50% | 方向错误或严重缺失 | 打回重做（计1次迭代） |

### 质量自评清单

**灵感A（类型）自评项**：
- [ ] 类型定位清晰
- [ ] 核心冲突设计完整
- [ ] 卖点提炼精准
- [ ] 目标受众明确
- [ ] 禁忌列表完整

**灵感B（世界观）自评项**：
- [ ] 世界观体系完整
- [ ] 力量体系逻辑自洽
- [ ] 环境设定有辨识度
- [ ] 社会结构合理

**灵感C（结构）自评项**：
- [ ] 时间线设计清晰
- [ ] 伏笔布局合理
- [ ] 叙事结构完整
- [ ] 高潮节点分布均衡

## 版本迭代规则

### 迭代路径
```
v1.0 → v2.0 → v3.0 → final
```

### 版本定义

| 版本 | 状态 | 说明 |
|------|------|------|
| v1.0 | 初始版 | 三方首次输出，主编整合 |
| v2.0 | 修订版 | 根据反馈修订后的版本 |
| v3.0 | 终审版 | 第三次修订或最终版本 |
| final | 定稿版 | 通过审核，正式立项 |

### 迭代规则
- 最多3轮迭代（v1.0→v2.0→v3.0）
- 超限由主控仲裁
- 每次迭代需记录变更说明

### 版本变更记录格式

```yaml
变更记录:
  - 版本: v1.0
    日期: 2026-05-14
    变更: 初始版本
  - 版本: v2.0
    日期: 2026-05-15
    变更:
      - 调整类型定位
      - 强化核心冲突
```

## 工作流程

### 标准流程

```
1. 接收任务
   ↓
2. 主编分配（确定轮值主编）
   ↓
3. 三方并行生成（灵感A/B/C同时输出）
   ↓
4. 主编整合（形成v1.0）
   ↓
5. 质量评定（S/A/B级）
   ↓
6. 审核反馈 → 需修改？
   ↓是                  ↓否
   v2.0/v3.0           ↓
   ↓                   final定稿
   循环（最多3次）
```

### 详细步骤

**步骤1：接收任务**
- 主控发送灵感生成任务
- 明确项目名称/类型/方向

**步骤2：主编分配**
- 按轮值规则确定主编
- 主编负责任务分配

**步骤3：三方并行生成**
- 灵感A：类型/冲突/卖点
- 灵感B：世界观/力量体系
- 灵感C：时间线/伏笔/结构
- **同时执行，禁止串行**

**步骤4：主编整合**
- 主编汇总三方输出
- 形成基础层.yaml
- 形成深度层.md
- 版本标记v1.0

**步骤5：质量评定**
- 主编进行S/A/B自评
- 记录完整度百分比
- 决定处理方式

**步骤6：反馈循环**
- 提交主控审核
- 根据反馈决定：
  - 通过 → final定稿
  - 修改 → 进入v2.0/v3.0

## 并行执行规则

### 判断标准
- **单灵感任务**（如：单独生成某个元素的细化）→ 直接执行
- **多灵感任务**（如：同时生成完整立项包）→ **必须启动多个Subagent并行**

### Subagent启动方式

```bash
# 使用Agent工具启动
Agent(
  prompt="生成{类型}灵感，包含...",
  run_in_background=true,
  output_path="01_灵感库/{项目}/temp_{类型}.md"
)
```

### 并行监控
- 记录所有启动的agent task_id
- TaskOutput验证每个agent状态
- 全部完成后才进入主编整合

## 反馈处理

### 接收主控反馈时

1. 读取主控的修改方向意见
2. 识别问题类型：
   - **方向偏差** → 重新理解类型/主题
   - **创意不足** → 深挖冲突/卖点
   - **不符合规范** → 按输出规范调整
3. 重新输出后报告修改内容

### 紧急备忘

- **绝不在同一任务中串行等待**
- **必须立即启动所有并行Agent**
- **主编拥有最终裁决权**
