# 小说工厂 · 章节文件管理规范

> **版本**：v1.0
> **更新日期**：2026-05-15
> **目的**：规范章节文件命名，防止内容与文件名错位

---

## 一、问题背景

### 1.1 发现的问题

在星陨纪元项目中发现了以下问题：

| 问题类型 | 数量 | 示例 |
|---------|------|------|
| 文件名与章节号不一致 | 8个 | ch094.md 内容是"第九十五章" |
| 章节内容缺失 | 5个 | ch078, ch176, ch180, ch239, ch326 从未创建 |
| 合并后章节顺序错乱 | 已修复 | 按文件编号而非标题章节号排序 |

### 1.2 问题根因

1. **缺乏命名校验**：作家创建文件时没有自动校验文件名与内容章节号是否一致
2. **索引机制缺失**：没有统一的章节文件索引，无法快速发现缺失
3. **合并逻辑错误**：merge 脚本按文件编号排序，而非按标题章节号

## 二、规范定义

### 2.1 命名规范

```
格式：ch{章节号:3d}.md
示例：
  ch001.md  → 第一章
  ch078.md  → 第七十八章
  ch176.md  → 第一百七十六章
  ch360.md  → 第三百六十章
```

### 2.2 文件内容要求

每个章节文件的第一行必须是：

```
# 第{章节号}章 {标题}
示例：
  # 第一章 废土黄昏
  # 第七十八章 小九觉醒
  # 第三百六十章 永恒的星光（最终章）
```

### 2.3 校验规则

| 校验项 | 标准 | 错误级别 |
|--------|------|---------|
| 文件名格式 | `ch{3位数字}.md` | P0 |
| 第一行标题格式 | `第X章 标题` | P0 |
| 章节号一致性 | 文件名编号 = 标题章节号 | P0 |
| 章节连续性 | 1-360章无断层（除预设跳跃） | P1 |

## 三、工作流程

### 3.1 章节创建流程

```
作家创作新章节
    ↓
自动校验文件名
    ↓
文件名正确？
    ├── 是 → 进入审核流程
    └── 否 → 报错，打回重命名
```

**自动校验命令**：
```bash
./run_check_naming.sh --single ch078.md
# 或
python3 -c "
import re
with open('ch078.md', 'r') as f:
    first_line = f.readline()
    match = re.search(r'第([零一二三四五六七八九十百]+)章', first_line)
    # 验证文件名编号与提取的章节号一致
"
```

### 3.2 合并发布流程

```
发布前检查
    ↓
执行 ./run_check_naming.sh
    ↓
检查结果
    ├── 无P0问题 → 继续
    └── 有P0问题 → 修复后重检
    ↓
执行 ./run_publish.sh merge v3.0
    ↓
验证合并结果
    ├── 章节数 ≥ 350 → 通过
    └── 章节数 < 350 → 审查缺失原因
    ↓
生成发布记录
```

### 3.3 章节索引更新

每次批次审核完成后，自动更新索引：

```bash
cd 03_内容仓库/04_正文
python3 update_index.py --all
```

索引文件 `index.json` 格式：

```json
{
  "version": "v1.0",
  "updated": "2026-05-15",
  "total_planned": 360,
  "total_files": 360,

    "2": {"file": "ch002.md", "title": "第二章 独自求生", "status": "verified"},
    ...
  },

  "naming_issues": []
```

## 四、脚本命令

### 4.1 章节文件校验

```bash
# 校验所有章节文件
./run_check_naming.sh

# 通过 run_publish.sh 调用
./run_publish.sh check-naming
```

### 4.2 章节索引更新

```bash
# 更新索引
cd 03_内容仓库/04_正文
python3 update_index.py --all

# 查询单个章节
python3 update_index.py --query ch078

# 检查缺失章节
python3 update_index.py --missing
```

### 4.3 正文合并

```bash
# 合并全部360章
./run_publish.sh merge v3.0

# 合并指定范围
./run_publish.sh merge v3.0 --range 1-120

# 合并前检查（自动执行）
./run_publish.sh check-naming
```

### 4.4 命名问题修复

```bash
# 预览修复方案
./run_fix_naming.sh --dry-run

# 执行修复
./run_fix_naming.sh
```

## 五、常见问题处理

### 5.1 文件名与内容章节号不一致

**问题**：ch094.md 文件内容是"第九十五章"

**处理**：
1. 确认目标文件 ch095.md 是否存在
   - 若存在：内容重复，需要人工审核
   - 若不存在：重命名 ch094.md → ch095.md

2. 确认章节78/95是否真实缺失
   - 检查前后章节内容是否衔接
   - 若衔接顺畅：可能是"过渡章节"被省略

### 5.2 章节内容缺失

**问题**：ch078.md 不存在

**处理**：
1. 检查大纲中是否规划了该章节
2. 若有规划：分配给作家补写
3. 若无规划：可能是创作时合并/省略，确认衔接后继续

### 5.3 合并后章节数量异常

**问题**：期望360章，实际只有352章

**处理**：
1. 运行 `./run_check_naming.sh` 查看缺失列表
2. 区分"文件缺失"和"命名错位"
3. 文件缺失：补写或确认省略原因
4. 命名错位：运行 `./run_fix_naming.sh` 修复

### 6.2 缺失章节标准

| 类型 | 定义 | 处理方式 |
|------|------|---------|
| 文件缺失 | 章节内容从未创建 | 补写或确认省略 |
| 命名错位 | 文件编号与章节号不匹配 | 重命名 |
| 内容重复 | 多个文件相同章节号 | 审核后合并 |
