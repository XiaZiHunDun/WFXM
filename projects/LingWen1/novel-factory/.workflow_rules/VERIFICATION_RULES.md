# 修复验证规则

> **目的**：防止"虚假修复"和"自我验证"，确保每批次审核和修复可追溯、可验证
>
> **核心原则**：验证者不能是修复者，必须由独立Agent执行验证

---

## 一、验证机制

### 1.1 验证脚本

| 脚本 | 功能 |
|------|------|
| `run_verify.sh` | Shell入口，提供check/sample/batch命令 |
| `run_verify_engine.py` | Python验证引擎，执行实际检查逻辑 |

### 1.2 验证类型

| 类型 | 触发条件 | 抽样比例 |
|------|---------|---------|
| **P0全量验证** | 发现P0问题时 | 100% |
| **P1抽样验证** | 发现P1问题时 | 30% |
| **随机抽样验证** | 每批次完成后 | 10% |
| **二次验证** | 修复后24小时内 | 100% |

### 1.3 验证时机

```
修复完成后 → 独立验证Agent执行验证 → 验证通过 → 才能标记完成
                                          ↓
                                    验证失败 → 重新修复
```

## 二、验证检查项

### 2.1 重复内容检查

| 检查点 | 方法 | 严重程度 |
|--------|------|---------|
| 章间重复 | 对比相邻章节前10行 | P0 |
| 章内重复 | 检测相同段落出现2次以上 | P0 |

**判定标准**：
- 如果章节开头3行以上与其他章节相同 → P0问题
- 如果同一段落在章内重复2次以上 → P0问题

### 2.2 章节编号检查

| 检查点 | 方法 | 严重程度 |
|--------|------|---------|
| 文件名与标题不匹配 | 提取标题中的章节号，与文件名对比 | P0 |
| 内容与章节号不匹配 | 检查内容中的章节引用是否一致 | P1 |

**示例**：
```
文件: ch291.md
标题: # 第二百九十三章 电磁奇点（上）
问题: 文件名是291，但标题是293章 → P0
```

### 2.3 叙事跳跃检查

| 检查点 | 方法 | 严重程度 |
|--------|------|---------|
| 场景突然转换 | 检查是否有过渡标记（——、与此同时等） | P1 |
| 时间线矛盾 | 对比前后章节时间线 | P0 |

### 2.4 角色一致性检查

| 检查点 | 方法 | 严重程度 |
|--------|------|---------|
| 代词不一致 | 统计"他/她/它"使用次数 | P1 |
| 形态描述矛盾 | 检查同一角色的不同形态描述 | P0 |

**示例**：
```
小九使用"它"(5次)和"她"(3次)不一致 → P0
```

## 三、验证执行流程

### 3.1 单章验证流程

```
1. 主控指派验证Agent（非原修复者）
2. 验证Agent读取源文件
3. 验证Agent执行 run_verify_engine.py check chXXX
4. 验证Agent读取验证报告
5. 验证Agent确认问题是否已修复
6. 生成最终验证结论
7. 更新 workflow_state.json
```

### 3.2 批次验证流程

```
1. 批次修复完成后，触发验证
2. 执行 run_verify_engine.py sample（随机36章）
3. 对P0问题章节执行100%验证
4. 生成验证报告
5. 通过率 < 80% → 打回重新修复
6. 通过率 >= 80% → 进入下一批次
```

### 3.3 二次验证流程

```
1. 修复完成后24小时内，执行二次验证
2. 验证Agent随机抽取20%已修复内容
3. 验证修复是否引入新问题
4. 验证修复是否真实生效（不能是表面修复）
```

## 四、验证状态追踪

### 4.1 验证日志

验证结果记录在 `.verify_log.json`：

```json
{
    "timestamp": "2026-05-18 12:00:00",
    "verified_chapters": ["ch291", "ch293", "ch295"],
    "sample_type": "p0",
    "results": {
        "ch291": "FAILED",
        "ch293": "FAILED",
        "ch295": "PASSED"
    }
}
```

### 4.2 验证报告

报告输出到 `06_意见仓库/04_正文_审核/`：

```
验证报告_sample.md    # 随机抽样验证
验证报告_p0.md       # P0全量验证
ch291_验证报告.md    # 单章验证
```

### 4.3 验证结论格式

```markdown
## 验证结论

| 章节 | 问题 | 验证状态 | 说明 |
|------|------|---------|------|
| ch291 | 章间重复 | ❌ 未修复 | 与ch293前10行相同 |
| ch293 | 章间重复 | ❌ 未修复 | 与ch291前10行相同 |
| ch295 | 编号错误 | ✅ 已修复 | 文件名与标题一致 |
```

## 五、禁止行为

```
❌ 禁止：修复者自己验证自己的修复
❌ 禁止：跳过验证直接标记完成
❌ 禁止：批量"无问题通过"但无抽检
❌ 禁止：修复速度过快（>10章/分钟）
❌ 禁止：验证报告内容与实际不符
```

---

## 六、通过标准

| 问题等级 | 通过标准 |
|----------|----------|
| P0 | 必须100%修复验证 |
| P1 | 通过率 >= 80% |
| P2 | 建议修复，不阻塞 |

**整体通过率** < 80% → 必须打回重新修复

## 七、命令速查

```bash
# 单章验证

# 随机抽样验证（10%）

python3 run_verify_engine.py sample

# P0全量验证
python3 run_verify_engine.py p0

# 批量验证
./run_verify.sh batch B01

# 查看验证状态
./run_verify.sh status
```
