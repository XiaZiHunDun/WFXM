## 主编轮值规则

### 轮值顺序
```
批次1 → 审核员A
批次2 → 审核员B
批次3 → 审核员C
批次4 → 审核员D
批次5 → 审核员E
批次6 → 审核员A（循环）
```

### 主编职责
- 汇总多方审核意见
- 裁决最终审核结论
- 质量等级评定（S/A/B）
- 版本管理与定稿标记

### 轮值限制
- F-J为待命审核员
- 主编缺席时由待命组替补
- 连续2批次后强制休息1批次

```
批次分配示例（ch001-ch010）：
┌─────────────────────────────────────────────┐
│ 第一组（ch001-ch005）                       │
│   审核员A（逻辑）+ 审核员B（人设）           │
│   + 审核员C（节奏）+ 审核员F（伏笔）         │
│   + 审核员J（综合）                          │
├─────────────────────────────────────────────┤
│ 第二组（ch006-ch010）                       │
│   审核员D（市场）+ 审核员E（设定）           │
│   + 审核员G（合规）+ 审核员H（风格）         │
│   + 审核员I（敏感度）                        │
└─────────────────────────────────────────────┘
```

### 并行调度原则
- 每批5人同时审核
- 同一章节最多2人重叠审核
- 重大问题需2人以上独立确认

## 质量评估等级

| 等级 | 完整度 | 处理方式 |
|------|--------|----------|
| **S级** | >90% | 直接进入下一环节 |
| **A级** | 70%-90% | 主编小幅调整后通过 |
| **B级** | 50%-70% | 返回作家修改（不计入迭代） |
| **不合格** | <50% | 打回重做（计1次迭代） |

### 评分计算
```
综合得分 = 逻辑×0.2 + 人设×0.15 + 节奏×0.15 + 市场×0.2 + 设定×0.1 + 伏笔×0.1 + 合规×0.05 + 风格×0.05
```

## 审核流程

### 标准流程
```
1. 接收任务 → 2. 技术审核 → 3. 市场审核 → 4. 意见汇总 → 5. 主编裁决 → 6. 定稿判定
```

### 详细步骤

#### 步骤1：接收任务
- 主控分配审核章节范围
- 审核员领取任务并记录
- 准备审核材料

#### 步骤2：技术审核（并行）
- 审核员A/E/F/B 执行技术层面检查
- 记录问题到意见仓库
- 标记问题优先级P0-P3

#### 步骤3：市场审核（并行）
- 审核员C/D/G/H/I 执行市场层面检查
- 记录问题到意见仓库
- 评估市场价值

#### 步骤4：意见汇总
- 主编收集所有审核意见
- 去重合并相同问题
- 按优先级排序

#### 步骤5：主编裁决
- 主编评审意见
- 给出修改建议
- 评定质量等级

#### 步骤6：定稿判定
- 通过：意见≤5条且无P0问题 → 标记`finalized`
- 需修改：启动作家修复 → 重新审核
- 死锁：超迭代上限 → 人工仲裁

## 双轨审核体系

### 技术审核 + 情感审核

小说工厂采用双轨审核体系，从两个维度全面检测作品质量：

| 审核类型 | 检查重点 | 审核员 | 输出目录 |
|---------|---------|--------|---------|
| **技术审核** | 逻辑/结构/设定冲突 | 审核员A-J | `06_意见仓库/04_正文_审核/` |
| **情感审核** | 情感弧线/配角价值/反派动机 | 审核员K（情感专项） | `06_意见仓库/情感审核/` |

### 情感审核专项

情感审核专项专门检测情感层面的质量问题，弥补技术审核的盲区。

#### 5项检查维度

| 维度 | 检查内容 | 适用阶段 |
|------|---------|---------|
| 情感弧线 | 主角/女主情感转折、婚后日常、情感锚点 | 大纲+正文 |
| 配角价值 | 非战力价值、独立支线、道具情感承载 | 大纲+正文 |
| 反派动机 | 反派前身、行动逻辑、验证者vs毁灭者 | 大纲审核 |
| 节奏健康度 | 阶段变速、核心冲突、信息密度 | 大纲审核 |
| 转折合理性 | 隐瞒必要性、理解铺垫、说服场景 | 大纲+正文 |

#### 调度时机

| 工作流步骤 | 启用审核专项 |
|-----------|------------|
| STEP_07 卷大纲审核 | 反派动机 + 节奏健康度 |
| STEP_11 阶段大纲审核 | 情感弧线 + 转折合理性 |
| STEP_16 正文审核 | 配角价值 + 情感锚点 |

#### 质量等级

| 等级 | 条件 | 处理 |
|------|------|------|
| S级 | 5项全部✅ | 直接进入下一环节 |
| A级 | 4项✅ + 1项⚠️ | 修复后复查 |
| B级 | 3项✅ + 2项⚠️ 或 有❌ | 返回修改（不计入迭代） |
| 不合格 | ≥3项❌ | 打回重做（计1次迭代） |

### 双轨协同流程

```
1. 主控调度：同时启动技术审核 + 情感审核
2. 并行执行：技术审核员(A-J) + 情感审核员(K) 并行工作
3. 报告合并：两份报告汇总到主控
4. 综合判定：技术问题≤5条 + 情感5项≥3✅ → 通过
            否则 → 返回修改
```
