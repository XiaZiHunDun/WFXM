# 汇总部门工作流程

## 概述

汇总部门负责将灵感、作家、审核各部门的工作成果整合为连贯的阶段/卷/全文汇总文档。

---

## 三角色分工详解

### 1. 汇总主笔

| 属性 | 说明 |
|------|------|
| 核心职责 | 汇总文档的核心整合与写作 |
| 调度时机 | 卷汇总需微调时、阶段/卷/全文汇总初稿 |
| 输出物 | 汇总文档初稿 |

**执行流程：**
1. 接收主控调度任务（携带问题清单）
2. 读取所有子内容（阶段汇总→卷汇总→全文汇总）
3. 脚本辅助提取关键事件清单
4. 人工整合：拼接+过渡撰写
5. 输出完整汇总文档
6. 输出修复报告到意见仓库
7. 等待主控验证

**禁止行为：**
- 不得直接修改文件（必须通过Agent执行）
- 不得跳过主控验证直接标记完成

---

### 2. 汇总编辑

| 属性 | 说明 |
|------|------|
| 核心职责 | 过渡润色和风格统一 |
| 调度时机 | 主笔完成修复后 |
| 输出物 | 编辑审核报告、二次润色稿 |

**执行流程：**
1. 接收主控调度（携带修复文件）
2. 读取修复后的文件
3. 审核过渡和风格
4. 输出编辑意见到意见仓库
5. 如需二次润色，执行后输出

**审核标准：**
- 风格统一检查：叙事口吻、用词习惯、句式结构
- 过渡段落润色：场景衔接、情绪过渡、时间跳跃
- 语气一致性：人物对话风格、旁白叙述语气

**禁止行为：**
- 不得跳过验证直接标记完成

---

### 3. 汇总校验

| 属性 | 说明 |
|------|------|
| 核心职责 | 人物/时间线一致性核查 |
| 调度时机 | 编辑通过后 |
| 输出物 | consistency_report.md、校验报告 |

**执行流程：**
1. 接收主控调度
2. 读取汇总文件全文
3. 执行一致性检查
4. 输出 consistency_report.md
5. 标记通过/需修改

**禁止行为：**
- 不得跳过校验直接发布

---

## 汇总执行标准流程

### 流程图

```
阶段汇总完成 → 卷汇总主笔 → 卷汇总编辑 → 卷汇总校验 → 主控确认 → 卷汇总定稿
                                       ↓
                               发现问题 → 返回主笔修复 → 重新编辑 → 重新校验
```

### 分层汇总顺序

| 层级 | 汇总内容 | 执行角色 |
|------|---------|---------|
| 阶段汇总 | 单个阶段内所有章节 | 汇总主笔（初稿）→ 编辑（润色）→ 校验（检查） |
| 卷汇总 | 单卷内所有阶段 | 汇总主笔（整合）→ 编辑（润色）→ 校验（检查） |
| 全文汇总 | 所有卷 | 汇总主笔（整合）→ 编辑（润色）→ 校验（检查） |

### 各层级汇总内容要求

**阶段汇总必须包含：**
- 阶段名称与编号
- 核心事件清单（按时间顺序）
- 人物状态变化
- 境界/能力进展
- 伏笔与回收记录
- 阶段小结

**卷汇总必须包含：**
- 卷名与卷号
- 各阶段核心事件
- 卷内人物关系图谱
- 卷时间线
- 核心冲突与解决
- 卷末状态

**全文汇总必须包含：**
- 作品名称
- 各卷核心脉络
- 全文时间线
- 人物命运总览
- 主题与寓意
- 世界观演变

## 汇总质量检查清单

### 汇总主笔自检清单

- [ ] 关键事件无遗漏
- [ ] 事件时间顺序正确
- [ ] 人物出场顺序合理
- [ ] 阶段/卷衔接自然
- [ ] 伏笔标注清晰

### 汇总编辑检查清单

- [ ] 叙事口吻统一
- [ ] 过渡段落流畅
- [ ] 无病句错句
- [ ] 标点符号规范
- [ ] 专有名词一致

### 汇总校验检查清单

- [ ] 人物名字无错
- [ ] 人物关系无矛盾
- [ ] 时间线无冲突
- [ ] 境界体系一致
- [ ] 地点设置无矛盾
- [ ] 跨卷引用一致

## 问题识别与上报机制

### 问题分级

| 级别 | 定义 | 处理方式 |
|------|------|---------|
| P0 | 逻辑矛盾、时间线冲突 | 立即上报主控，返回主笔修复 |
| P1 | 人物状态不一致、境界描述错误 | 上报主控，安排修复 |
| P2 | 风格不统一、过渡不流畅 | 交由编辑处理 |
| P3 | 错别字、标点错误 | 编辑直接修正 |

### 上报流程

```
发现问题 → 判断级别 → P0/P1上报主控 → 主控调度修复 → 重新走校验流程
                              → P2交编辑 → 编辑处理 → 编辑自审
                              → P3编辑直接修正
```

### 问题记录格式

```markdown
## 问题报告

**文件：** 卷1_阶段1_汇总.md
**级别：** P0
**类型：** 时间线冲突
**描述：** 第三章提到"星陨历47年"，但第五章提到"星陨历45年"
**影响范围：** 卷1全卷
**建议修复：** 统一为"星陨历47年"
```

| 验证汇总完整性 | `./run_summary.sh verify` |

## 正文合并命令（v2.0 新增）

汇总部门负责将各层级章节文件合并为完整的正文合并文件。

| 合并层级 | 命令 | 输出位置 |
|---------|------|---------|
| 阶段正文 | `./run_summary.sh merge stage 卷1_阶段1` | `07_汇总仓库/阶段正文/` |
| 卷正文 | `./run_summary.sh merge volume 卷1` | `07_汇总仓库/卷正文/` |
| 全文正文 | `./run_summary.sh merge full` | `07_汇总仓库/全文正文/` |

**合并层级说明：**

```
阶段正文 (ch001-ch020)     ──┐
                           ├──> 卷正文 (ch001-ch140)  ──┐
阶段正文 (ch021-ch040)     ──┤                           │
                           │                           ├──> 全文正文
阶段正文 (ch041-ch060)     ──┤                           │
...                        ├──> 卷正文 (ch141-ch240)  ──┤
                           │                           │
阶段正文 (ch121-ch140)     ──┘                           │
                                                        ├──> (ch241-ch360)
阶段正文 (ch141-ch160)     ──┐                           │
...                        ├──> 卷正文 (ch241-ch360)  ──┘
阶段正文 (ch321-ch360)     ──┘
```

**合并流程：**
1. 汇总主笔完成各层级汇总文档（阶段→卷→全文）
2. 使用 merge 命令合并对应层级正文
3. 汇总编辑审核合并后的正文
4. 汇总校验检查一致性
5. 主控确认后定稿

## 仓库职责划分

| 仓库 | 职责 | 内容 |
|------|------|------|
| **03_内容仓库** | 原始素材库 | 大纲文件 + 章节源文件 |
| **07_汇总仓库** | 汇总产出库 | 正文合并文件（按层级）+ 汇总报告 + 汇总工作文件 |
| **08_已发布** | 正式发布库 | 最终定稿版本 + 发布记录 |

**07_汇总仓库 目录结构：**
```
07_汇总仓库/
├── 阶段正文/       ← 17个阶段正文合并文件
├── 卷正文/         ← 3个卷正文合并文件
├── 全文正文/       ← 全文正文合并文件
├── 汇总报告/       ← 汇总报告、大纲、修改记录
├── 汇总主笔/       ← 汇总主笔工作区
├── 汇总编辑/       ← 汇总编辑工作区
└── 汇总校验/       ← 汇总校验工作区
```

**08_已发布 目录结构：**
```
08_已发布/
├── 发布管理工作流程.md
├── 归档/           ← 历史归档
└── 历史版本/       ← 旧版本存档
```
