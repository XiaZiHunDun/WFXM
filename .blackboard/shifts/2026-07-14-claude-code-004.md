### Runner 修复（2 个 bug）

1. **多违例解析** — Python 端把 `violated` 列表 join 成三字符 `'|||'`，bash `tr '|||' '\n'` 把它当字符类只产生两个换行，下游 `cut -d'|' -f3` 拿首段，2/2 fixture 被算成 1。RED 已复现 `old_parser=[CA4] expected=[CA4,T8]`。
   - 改法：Python 改用 `os.environ['PILOT_VIOLATIONS']` 注入；Python 输出整 JSON（含 sorted(set(violated))）；bash 用三个独立 `python3 -c` 分别解析 `ok` / `issues` / `violated`，避免任何 shell 层切词。
2. **缺 cleanup trap** — 原脚本 `set -uo pipefail`（无 -e）+ 没有 trap；Ctrl-C / SIGTERM / python rc!=0 都可能让 strict 留在 `on`。HIGH 复审发现后补 `cleanup_strict` + `trap cleanup_strict EXIT` + `trap 'exit 130' INT` + `trap 'exit 143' TERM`；末段显式 cleanup 后 `trap -` 避免重复。
