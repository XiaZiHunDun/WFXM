## 工作内容

### P0: 智能工具选择 + 执行优化

**智能工具推荐集成** (`butler/core/agent_loop_phases.py`)
- 在 `_phase_enrich_user_text()` 中集成 `ToolService.recommend_tools()`
- 基于经验+语义+分类的综合推荐，置信度阈值 0.3
- 推荐结果与 skill_preferred_tools 合并作为首选工具

**工具执行优化** (`butler/core/agent_loop.py`)
- 修改 `_dispatch_tool()` 使用 ToolService 执行
- 支持缓存、去重、性能监控
- 失败时自动降级到原有直接调用

**ToolService 增强** (`butler/tools/tool_service.py`)
- 移除 handler 限制，直接通过 registry 调用工具
- 应用缓存和去重优化到所有工具调用

### P1: 经验注入 + 经验写入

**预回合经验注入** (`butler/core/agent_loop.py`)
- 在 `_build_turn_ephemeral_system()` 中添加经验树检索逻辑
- 从最近5个回合的 user_intent 构建查询
- 调用 `ExperienceRetriever.retrieve("default", query, top_k=3)` 获取相关经验
- 置信度 > 0.3 的经验注入到 ephemeral_system
- 格式：`[经验 {source} 得分={score}]: {content}`

**对话结束经验写入** (`butler/core/agent_loop.py`)
- 在 `_update_conversation_state()` 中添加经验写入逻辑
- 只在有工具调用时写入，避免无效经验积累
- 写入成功的工具调用：`query -> tool_name` + result
- 限制最多写入前5个工具调用

### P2: 语义感知上下文压缩

**关键词提取** (`butler/core/context_compressor.py`)
- 新增 `_extract_keywords_from_latest_user()` 函数
- 从最新用户消息中提取 3 个字符以上的中文/英文单词
- 自动提取最多 10 个关键词

**语义保护拆分** (`butler/core/context_compressor.py`)
- 修改 `_split_head_tail()` 添加 `protect_keywords` 参数
- 扫描 middle 区域消息，发现包含保护关键词的消息时，扩展 tail 区域保护这些消息
- 避免相关上下文被错误总结

**压缩管道集成** (`butler/core/context_compress_pipeline.py`)
- 修改 `_split_for_compaction()` 和 `run_compress_messages()` 支持 `protected_keywords` 参数

- **工具推荐**: 基于语义匹配返回工具推荐列表（5个）
- **经验写入**: 成功写入经验到经验树
- **经验检索**: 从经验树检索相关经验（3个结果）

## 新增诊断指标

- `experience_injected`: 本回合注入的经验数量
- `experience_written`: 本回合写入的经验数量
- `semantic_protection_keywords`: 用于语义保护的关键词数量

## 关键设计决策

| 决策 | 理由 |
|------|------|
| **Lazy Import** | 在函数内部延迟导入，避免循环依赖 |
| **safe_best_effort** | 新增逻辑失败不影响主流程，自动降级 |
| **工具调用门控** | 只在有工具调用时写入经验，避免噪声 |
| **轻量级关键词匹配** | 避免使用 embedding/ChromaDB，保持压缩性能 |
