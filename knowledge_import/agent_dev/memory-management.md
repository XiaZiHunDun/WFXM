# Agent记忆管理

## 记忆分层架构

### 热层（Ephemeral）

存储最近N轮对话内容，用于保持对话连贯性。

**特点：**
- 实时访问，低延迟
- 有限容量，滚动更新
- 适合短期上下文

**实现策略：**
- 固定大小的队列
- 超出容量时移除最早的记录
- 保留关键信息（工具调用、重要决策）

### 温层（Chapter）

每M轮对话生成一次章节摘要，用于中期记忆。

**特点：**
- 结构化摘要，便于检索
- 中等容量，定期更新
- 适合任务级上下文

**实现策略：**
- 每10轮生成一次摘要
- 提取关键技术点和决策
- 存储到向量数据库

### 冷层（Semantic）

基于语义检索的长期记忆，用于知识存储和检索。

**特点：**
- 大容量，持久存储
- 语义相似度检索
- 适合长期知识

**实现策略：**
- 使用向量数据库（如ChromaDB）
- 支持语义相似度查询
- 定期更新和维护

## 记忆检索策略

### 1. 语义检索

基于向量相似度查找相关记忆：
- 计算查询向量与存储向量的相似度
- 返回Top-K最相似的结果
- 适用于模糊查询和联想记忆

### 2. 关键词检索

基于关键词匹配查找记忆：
- 使用FTS（全文搜索）索引
- 支持精确匹配和模糊匹配
- 适用于明确的信息查找

### 3. 图检索

基于实体关系查找记忆：
- 构建知识图谱
- 支持多跳路径查找
- 适用于关系推理

### 4. 混合检索

融合多种检索方式：
- 先语义检索缩小范围
- 再关键词检索精确定位
- 最后图检索扩展关联

## 记忆优化技巧

### 上下文压缩

- 删除冗余信息
- 生成对话摘要
- 保留关键决策点

### 记忆去重

- 基于内容哈希去重
- 合并相似记忆
- 更新过时记忆

### 记忆优先级

- 根据使用频率调整优先级
- 近期使用的记忆优先保留
- 重要的记忆标记为高优先级

## 记忆持久化

### 存储方案

- **SQLite**：轻量级，适合本地存储
- **PostgreSQL**：支持全文搜索和扩展功能
- **ChromaDB**：向量存储，适合语义检索
- **Redis**：缓存热层记忆，提高访问速度

### 备份策略

- 定期备份数据库
- 增量备份变化数据
- 异地备份防止数据丢失

## 记忆评估指标

### 定量指标

- **召回率**：相关记忆被检索到的比例
- **准确率**：检索结果的相关性
- **延迟**：检索响应时间
- **覆盖率**：记忆覆盖的知识范围

### 定性评估

- **对话连贯性**：多轮对话是否保持一致
- **知识复用**：是否能正确使用历史知识
- **错误纠正**：是否能从错误中学习

## 常见问题

### 记忆混淆

**原因**：不同任务的记忆相互干扰

**解决方案**：
- 按任务/会话隔离记忆
- 使用命名空间区分不同上下文
- 在记忆中添加元数据标签

### 记忆过时

**原因**：知识更新不及时

**解决方案**：
- 定期检查和更新记忆
- 添加时间戳和版本信息
- 标记过时知识并提示更新

### 记忆膨胀

**原因**：记忆无限增长

**解决方案**：
- 设置记忆容量上限
- 定期清理不常用的记忆
- 使用压缩算法减少存储占用

## 总结

记忆管理是Agent能力的核心，需要合理设计分层架构和检索策略。通过不断优化和评估，可以提高Agent的知识复用能力和对话质量。
