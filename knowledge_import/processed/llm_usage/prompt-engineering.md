# 提示词工程指南

## 核心原则

### 1. 明确性原则

清晰描述任务，避免模糊表述：

**不好**：
```
写一段关于人工智能的文章
```

**好**：
```
写一篇500字左右的科普文章，介绍人工智能的基本概念、发展历程和应用场景。目标读者是高中学生，语言通俗易懂，避免专业术语。
```

### 2. 结构化原则

使用分点列表、标记分隔和代码块：

```
请按照以下格式回答：

## 问题分析
[分析问题的关键点]

## 解决方案
1. [方案一]
2. [方案二]

## 代码示例
```python
[代码实现]
```
```

### 3. 示例原则

提供输入输出示例，使用few-shot学习：

```
请将以下句子翻译成中文：

示例1：
输入：Hello, world!
输出：你好，世界！

示例2：
输入：Artificial intelligence is changing the world.
输出：人工智能正在改变世界。

现在翻译：
输入：Machine learning algorithms can learn from data.
输出：
```

### 4. 角色原则

设定明确角色，定义专业背景：

```
你是一位资深的Python程序员，拥有10年开发经验。请用专业但易懂的语言回答以下问题：

[问题]
```

### 5. 迭代原则

测试提示词效果，逐步优化：

1. 初步测试：检查基本功能是否正常
2. 分析失败案例：找出问题所在
3. 针对性优化：调整提示词结构和内容
4. 重新测试：验证优化效果

```

2. 每个子任务清晰明确
3. 按优先级排序

```

```

```

## 进阶技巧

### 链式思考

引导模型进行逐步推理：

```
请解决以下数学问题，写出详细的思考步骤：

问题：[问题描述]

思考步骤：
1. 首先，我需要理解问题的核心是什么
2. 然后，分析已知条件和未知条件
3. 接下来，确定解决问题的方法
4. 最后，逐步计算得出答案

答案：
```

### 自我反思

让模型检查和修正自己的回答：

```
请先给出一个初步答案，然后检查是否有错误或遗漏：

问题：[问题描述]

初步答案：[模型生成的答案]

自我检查：
1. 答案是否完整？
2. 是否有逻辑错误？
3. 是否遗漏了重要信息？

修正后的答案：
```

### 多视角分析

从多个角度分析问题：

```
请从以下三个角度分析这个问题：

问题：[问题描述]

角度1：技术角度
[技术层面的分析]

角度2：业务角度
[业务层面的分析]

角度3：用户角度
[用户体验层面的分析]

综合分析：
```

## 参数调优

### Temperature

控制输出的随机性：
- **0.0**：确定性输出，适合需要精准答案的场景
- **0.3-0.5**：平衡创意和一致性，适合大多数场景
- **0.7-1.0**：高创意输出，适合生成性任务

```python
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": prompt}],
    temperature=0.7
)
```

### Top_p

控制输出的多样性：
- **0.9**：常用设置，平衡多样性和相关性
- **0.5**：更集中，适合需要精准答案的场景
- **0.95**：更多样化，适合创意生成

```python
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": prompt}],
    top_p=0.9
)
```

### Max_tokens

控制输出长度：
- 根据任务类型设置
- 避免浪费token
- 防止截断重要内容

```python
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": prompt}],
    max_tokens=1000
)
```

### Presence_penalty / Frequency_penalty

控制重复内容：
- **正数**：减少重复，鼓励新内容
- **负数**：增加重复，保持一致性

```python
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": prompt}],
    presence_penalty=0.1,
    frequency_penalty=0.1
)
```

## 常见错误

### 提示词过长

**问题**：提示词超过模型上下文限制

**解决方案**：
- 精简提示词
- 使用摘要代替完整文本
- 分批处理长文本

### 指令冲突

**问题**：提示词中包含相互矛盾的指令

**解决方案**：
- 检查并消除冲突
- 明确优先级
- 使用结构化格式

### 缺乏上下文

**问题**：模型缺少必要的背景信息

**解决方案**：
- 提供相关上下文
- 引用历史对话
- 使用检索增强生成(RAG)

### 格式要求不明确

**问题**：输出格式不符合预期

**解决方案**：
- 明确指定格式
- 提供格式示例
- 使用模板约束输出

## 评估方法

### 自动评估

使用指标评估提示词效果：
- **准确率**：答案的正确率
- **完整率**：是否覆盖所有要点
- **格式符合率**：是否符合指定格式
- **响应时间**：生成答案的时间

### 人工评估

邀请人类评估提示词效果：
- 评估答案质量
- 评估格式符合度
- 评估语言流畅度
- 收集反馈意见

### A/B测试

比较不同提示词的效果：
- 设计对照组和实验组
- 在相同条件下测试
- 统计分析结果
- 选择最优提示词

## 总结

提示词工程是发挥LLM能力的关键。通过遵循核心原则、使用模板和不断迭代优化，可以显著提升模型的输出质量和效率。
