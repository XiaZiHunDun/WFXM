**`ConnectionPool[T]`** — a thread-safe, generic connection pool with:

| Method | Description |
|---|---|
| `get(key)` | Retrieve a value; returns `None` if missing or expired (lazy eviction) |
| `put(key, value, ttl?, priority?, delay?)` | Store a value; when `delay > 0`, the write is scheduled as a priority task |
| `delete(key)` | Remove a key, returns `True` if it existed |
| `submit(func, ..., priority?, delay?, ttl?, key?)` | Schedule any callable for priority-based execution |
| `flush(max_tasks?)` | Execute all ready tasks in priority order |
| `evict()` | Purge all expired entries |
| `clear()` | Wipe all entries and pending tasks |

**Key design points:**
- **Priority queue** via `heapq` — tasks are ordered by `(priority, scheduled_at)`, so `CRITICAL` tasks run before `LOW`
- **Delayed scheduling** — `put(..., delay=2.0)` or `submit(..., delay=5.0)` enqueues work for the future
- **Automatic expiration** — each entry carries a TTL; expired entries are evicted lazily on `get()` or in bulk via `evict()`
- **Thread safety** — a `Lock` guards the store/heap, a `Semaphore` bounds concurrency to `max_connections`
- **Type hints** — `Generic[T]` propagates the value type through all public APIs
