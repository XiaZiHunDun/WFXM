Worth noting: this function is purely CPU-bound with no I/O, so making it async adds overhead without benefit. If you're calling it from an async context just to avoid blocking, consider running it in an executor instead:

```python
import asyncio

def calculate(data):
    total = 0
    for item in data:
        if item['type'] == 'A':
            total += item['value'] * 1.1
        elif item['type'] == 'B':
            total += item['value'] * 0.9
        elif item['type'] == 'C':
            total += item['value'] * 1.05
    return total

async def calculate_async(data):
    return await asyncio.get_event_loop().run_in_executor(None, calculate, data)
```

This keeps the sync function intact and offloads it to a thread pool, which is the proper pattern for CPU-bound work in async code.
