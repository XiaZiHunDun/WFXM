Both implementations are in `scc.py` and the results confirm correctness across all test sizes.

**Tarjan's algorithm** — single DFS pass, O(V + E). It maintains a stack of vertices currently in the DFS path and uses `lowlink` values to detect when an entire SCC has been explored.

**Brute-force** — O(V² · (V + E)). Computes full reachability sets via BFS from every vertex, then groups vertices that are mutually reachable using union-find.

**Performance comparison:**

| Vertices | Tarjan | Brute-force | Speedup |
|----------|--------|-------------|---------|
| 100      | 0.39ms | 6.71ms      | 17x     |
| 500      | 0.96ms | 153ms       | 160x    |
| 1000     | 1.91ms | 607ms       | 318x    |
| 2000     | 4.20ms | 2655ms      | 633x    |

The brute-force approach degrades roughly as O(V³) in practice (since edges grow proportionally), while Tarjan stays linear — the speedup grows with graph size.
