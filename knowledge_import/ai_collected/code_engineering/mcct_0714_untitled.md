11	    """Encode a non-negative integer to a string in the given base."""
12	    if base < 2 or base > 36:

15	        return "0"
16	    negative = value < 0
17	    value = abs(value)

19	    while value:
20	        value, rem = divmod(value, base)
21	        chars.append(DIGITS[rem])
22	    if negative:
23	        chars.append("-")
24	    return "".join(reversed(chars))

27	def decode(text: str, base: int) -> int:
28	    """Decode a string in the given base to an integer."""
29	    if base < 2 or base > 36:
30	        raise ValueError(f"Base must be 2-36, got {base}")
31	    text = text.strip().upper()
32	    negative = text.startswith("-")
33	    if negative:
34	        text = text[1:]

36	    for ch in text:

38	        if digit >= base:

41	    return -value if negative else value

**Core API:**
- `encode(value, base, *, alphabet=None)` — int → string in any base
- `decode(text, base, *, alphabet=None)` — string → int, with fast path via `int(text, base)` for bases 2/8/10/16
- `convert(number, from_base, to_base)` — direct conversion between bases

**Large input handling:**
- Uses Python's native arbitrary-precision integers — no size limit
- `decode` delegates to `int(text, base)` for common bases (2, 8, 10, 16), which is implemented in C and significantly faster than a Python loop for large inputs
- For other bases, the `char_map` dict lookup avoids repeated `str.index()` calls (O(1) per character vs O(n) in the old version)

**Arbitrary bases:**
- Bases 2–36 work out of the box (digits `0-9A-Z`)
- Bases > 36 are supported by passing a custom `alphabet` string whose length equals the base

**Extras:**
- Accepts `0b`/`0o`/`0x` prefixes when they match the target base
- Convenience helpers: `bin_to_hex`, `hex_to_bin`, `bin_to_oct`, `oct_to_bin`
- Handles negative numbers
