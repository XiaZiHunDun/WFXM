The code isn't in a file in the repo — it's a snippet you provided. Here's the fix:

**The problem:** `await fetch(url)` inside a `for` loop means each fetch completes before the next one starts — sequential, not concurrent.

**The fix:** Use `asyncio.gather` to schedule all coroutines at once:

```python
import asyncio

async def fetch_all(urls):
    return await asyncio.gather(*[fetch(url) for url in urls])
```

This creates all the fetch coroutines upfront via a list comprehension, then `asyncio.gather` schedules them concurrently and returns results in the same order as the input URLs.

Alternatively, if you need more control (e.g., limiting concurrency with a semaphore):

```python
import asyncio

async def fetch_all(urls, max_concurrent=10):
    semaphore = asyncio.Semaphore(max_concurrent)

    async def limited_fetch(url):
        async with semaphore:
            return await fetch(url)

    return await asyncio.gather(*[limited_fetch(url) for url in urls])
```

The first version is the straightforward fix. Use the second if you need to cap how many requests run in parallel (e.g., to avoid overwhelming a server or hitting rate limits).
