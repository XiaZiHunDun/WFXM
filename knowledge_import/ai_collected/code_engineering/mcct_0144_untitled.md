5	    """Disjoint Set Union with path compression and union by rank."""
6	
7	    def __init__(self, n):
8	        self.parent = list(range(n))
9	        self.rank = [0] * n
10	
11	    def find(self, x):
12	        if self.parent[x] != x:
13	            self.parent[x] = self.find(self.parent[x])
14	        return self.parent[x]
15	
16	    def union(self, x, y):
17	        rx, ry = self.find(x), self.find(y)
18	        if rx == ry:
19	            return False
20	        if self.rank[rx] < self.rank[ry]:
21	            rx, ry = ry, rx
22	        self.parent[ry] = rx
23	        if self.rank[rx] == self.rank[ry]:
24	            self.rank[rx] += 1
25	        return True

All tests pass. Here's what `prim.py` does:

**Algorithm** — standard lazy-Prim's with `heapq`:
1. Build an adjacency list from the edge list.
2. Push the start vertex with weight 0 onto a min-heap.
3. Pop the cheapest edge; skip if the target is already visited. Otherwise mark it visited, record the edge, and push all its neighbors.
4. Repeat until all `n` vertices are collected (or the heap empties, meaning the graph is disconnected).

**Edge cases handled:**
- `n == 0` or `n == 1` — returns `([], 0)` immediately.
- Disconnected graph — raises `ValueError` if fewer than `n-1` MST edges are found.
- Invalid vertex indices in edges or `start` — raises `ValueError`.
- Self-loops and parallel edges — naturally ignored (duplicate pushes are skipped by the `visited` check).

**Complexity:** O(E log V) time, O(V + E) space — same as the Kruskal implementation but using Prim's approach, which is often preferred when the graph is already represented as an adjacency list or is dense (E ≈ V²).
