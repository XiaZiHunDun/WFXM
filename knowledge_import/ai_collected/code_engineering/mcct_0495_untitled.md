Here's the fix: use a `with` statement so the file handle is always closed, even if an exception occurs.

```python
def read_file_lines(filepath):
    with open(filepath, 'r') as f:
        lines = f.readlines()
    return [line.strip() for line in lines]
```

The `with` statement calls `f.close()` automatically when the block exits, eliminating the resource leak.
