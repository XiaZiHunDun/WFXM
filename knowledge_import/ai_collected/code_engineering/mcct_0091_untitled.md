2	    """Find the longest palindromic substring using expand-around-center."""
3	    if len(s) < 2:
4	        return s
5	
6	    start, end = 0, 0
7	
8	    def expand(left: int, right: int) -> tuple[int, int]:
9	        while left >= 0 and right < len(s) and s[left] == s[right]:
10	            left -= 1
11	            right += 1
12	        return left + 1, right - 1
13	
14	    for i in range(len(s)):
15	        # Odd-length palindromes
16	        l1, r1 = expand(i, i)
17	        if r1 - l1 > end - start:
18	            start, end = l1, r1
19	        # Even-length palindromes
20	        l2, r2 = expand(i, i + 1)

All 14 tests pass. Here's what the `BloomFilter` class provides:

**Core operations:**
- `add(item)` — insert a string or bytes element
- `item in bf` — probabilistic membership check (`__contains__`)
- `len(bf)` — number of elements added
- `bool(bf)` — whether the filter is non-empty

**Advanced features:**
- `merge(other)` — union of two compatible filters
- `estimated_fpp()` — current false-positive probability based on fill level
- `to_bytes()` / `from_bytes()` — serialization/deserialization
- Capacity enforcement with `OverflowError` to protect the error rate

**Internals:**
- Optimal sizing via `m = -(n * ln(p)) / (ln2)^2` and `k = (m/n) * ln2`
- Double-hashing scheme (MD5 + SHA1) to generate `k` independent hash positions from just two digest computations
- No external dependencies (stdlib only: `hashlib`, `math`)
