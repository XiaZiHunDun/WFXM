## 可用模型参考

| 模型 | 可用 | 特点 | 适合场景 |
|------|------|------|---------|
| MiniMax-M2.7 | ✅ | 最新最强，延迟 ~6s | 管家层、复杂任务 |
| MiniMax-M2.5 | ✅ | 均衡，延迟 ~2s | 项目 Agent |
| MiniMax-M2.5-highspeed | ✅ | 快速版 M2.5 | 实时交互 |
| MiniMax-M2.1 | ✅ | 轻量，延迟 ~1.5s | 简单查询 |
| MiniMax-M2 | ✅ | 基础版 | 测试用途 |
| MiniMax-M2.7-highspeed | ⚠️ 受限 | 需更高级套餐 | — |
| MiniMax-M2.1-highspeed | ⚠️ 受限 | 需更高级套餐 | — |

模型切换命令:
- CLI: `/model <层> minimax:<模型名>`
- 微信: `/model <层> minimax:<模型名>`
- 层可选: `butler`, `dev`, `content`, `review`

## 微信网关技术说明

Butler 的微信网关基于腾讯 **iLink Bot API** 实现（与 hermes-agent 使用相同的协议），主要特点：

| 项目 | 说明 |
|------|------|
| 协议 | HTTPS JSON，长轮询 |
| 登录 | QR 码扫码（终端 ASCII + URL） |
| 收消息 | `ilink/bot/getupdates` 长轮询 |
| 发消息 | `ilink/bot/sendmessage` HTTP POST |
| 平台兼容 | Linux / macOS / Windows，无需桌面微信 |
| 凭证持久化 | `~/.butler/weixin/accounts/<id>.json` |
| 会话断线 | 自动重试 + 退避策略 |
| 消息去重 | message_id + 内容指纹双重去重 |
