## 一、方案里已有的设计（摘录）

| 设计点 | 出处 | 含义 |
|--------|------|------|
| **100+ 场景** | 模块设计 §七 | AgentLoop + LW-REAL 合计的**最低规模**参考（2026-05 初稿时 v1～v3 规划） |
| **槽位五元组** | v3 生成方案 §二 | `维度 × 意图 × 难度 × 问法 × 期望形态`，每批 **30～40 条** 单轮 + **2～3 组** 多轮 |

## 二、提交前规模目标（2026-05 修订）

### AgentLoop（`dev_assistant.*`）

| 指标 | 目标 | 说明 |
|------|------|------|
| **case 条目总数** | **≥ 180** | 含多轮算 1 case、组内 3 轮另计 |
| **单轮 case** | **≥ 150** | mock/live 参数化主体 |
| **多轮组** | **≥ 12** | 每组 ≥2 轮，测上下文 |
| **套件批次数** | **≥ 5** | v1 基础 + v2 深度 + v3 对话 + v4 盲区 + v5 补全 |
| **meta.dimensions 去重** | **≥ 40** | 跨套件维度并集（当前约 **50+**） |
| **每维至少** | **≥ 2 条** | 单轮或 1 组多轮 |

### Gateway（`wechat_real.*`）

| 指标 | 目标 | 说明 |
|------|------|------|
| **话术目录条数** | **≥ 55** | `utterance_catalog.yaml`（含 legacy 索引 + 可执行条目） |
| **可执行话术** | **≥ 45** | `test_gateway_utterance_catalog.py` 数据驱动 |
| **LW-REAL 手写场景** | **≥ 10** | `test_gateway_dev_conversations.py` 真机黄金路径 |
| **合计 gateway pytest** | **≥ 60** | 数据驱动 + LW-REAL + DEV 习惯 |

### 未达标前

- **不提交**语料大 PR（模块代码可另 PR，语料与文档同批提交更利于 review）  
- **不跑**全量 live（按你方要求）；mock 必须全绿

## 五、与「足够多」的判断

满足以下 **全部** 即可进入「统一 live + 提交」阶段：

1. AgentLoop case **≥ 180**  
2. 维度并集 **≥ 40**，且无「整维缺失」  
3. `pytest tests/corpus -m corpus_mock` 全绿  
4. `corpus-scale-target` 与 `registry.yaml` 一致  
5. 问题地图模板就绪（live 可后补）  

未要求：首轮 live 100% 通过（属优化阶段）。
