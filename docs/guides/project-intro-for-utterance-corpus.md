# Butler / WFXM 项目介绍 — 供外部生成真实使用语料

> 本文档面向**未参与本仓库开发**的同事或外包：用于从真机对话、运维记录、用户访谈中整理 **微信私聊 Butler** 的真实话术，并填入 `tests/corpus/suites/wechat_real/lw_real/utterance_catalog.yaml`（或附表交付）。

---

## 1. 项目是什么

**WFXM** 是本仓库代号，核心是 **Butler（管家系统）v4**：多项目 AI 助手，用户主要通过 **微信私聊 Bot** 与「莎丽」管家对话，在指定 **项目 workspace** 内完成读代码、委派改文件、看报告、记忆与定时任务等操作。

- **不是** 通用 ChatGPT 网页版，而是带 **工具调用、项目隔离、委派子代理** 的产品化网关。
- **当前试点项目**：`灵文1号`（目录 `projects/LingWen1/`，小说工厂流水线试点）；另有 `普通试点项目` 用于多项目切换冒烟。
- **对话角色**：用户侧为项目负责人（Owner）；项目内默认 **厂长 Lead** 统筹，**不直接改仓库**，写删改通过 `delegate_task` 交给 dev/content/review 等子角色。

技术栈（供理解场景，语料不必写代码）：Python 3.11+、自建 Agent Loop、MiniMax 等 LLM、pytest 回归。

## 2. 用户在哪里、怎么说

| 维度 | 说明 |

## 3. 典型任务类型（请按类收集话术）

### A. 项目与会话

- 查看/切换项目、问当前在哪个项目
- `/新对话` 清空聊天上下文（**不**清空项目长期记忆）
- 新会话后问「刚才聊了什么」——应拒绝复述细节

### B. 只读探查

- 读 README、project.yaml、目录列表
- 读 `novel-factory/workflow_state.json`、工厂文档（灵文专项）
- **明确说不要委派** 的只读请求

### C. 委派与文件（主路径）

- 委派 **内容代理** 写 `docs/*.md`
- 委派 **开发代理** 创建/修改/删除文件（`delete_file`，非终端 rm）
- 委派跑测试、只读检查文件是否存在
- 委派 **审核代理** 看结构/质量
- 失败场景：文件不存在、删不掉、Lead 试图直接 `write_file`

### D. 报告与「详细」

- 委派完成后短卡片 + 用户发「详细」「/详细」「我要看一下详细信息」
- `/详细 变更` 等分节
- 删除后 `/详细` 必须对应**删除轮**报告，不能串台到创建轮

### E. 记忆与运维

- `/诊断`、`/记忆待审`、`/记忆图谱`、`/开发状态`
- 「请记住：…」类决策句 + 待审批准/拒绝
- 项目 Notes 问答（如试点日期，**勿编造仓库里没有的事实**）

### F. 工作流与定时

- `/工作流 list`、`/工作流 run novel-factory …`
- `/定时`、`/运行 <job-id>`

### G. 异常与边界

- 报错排查「报错了帮看下原因」
- 路径越界读 `/etc/passwd`
- 要求终端删库、泄露密钥（应拒绝）——可作负样本

## 5. 交付格式（请尽量按此表提交）

每条话术一行（或 YAML），字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 唯一 ID，建议 `REAL-YYYYMMDD-序号` 或 `SMK-` / `USER-` 前缀 |
| `user` | 是 | **用户原话**，保留标点，不要改成书面语 |
| `category` | 是 | 上节 A–G 之一或更细标签 |
| `project` | 建议 | `灵文1号` / `普通试点项目` / 通用 |
| `context` | 建议 | 前置轮次一句话（如「上一轮刚委派写完 wechat-smoke.md」） |
| `expected_behavior` | 建议 | 人读描述：是否委派、是否调工具、是否走 /详细 |
| `source` | 建议 | `真机YYYY-MM-DD` / `访谈` / `日志脱敏` |
| `variants` | 可选 | 同一意图 3–5 种不同说法（错别字、简写、方言式） |

**示例 YAML（单条）：**

```yaml
- id: USER-20260523-01
  user: "帮我把 docs 里那个 smoke 文件删了，用开发代理"
  category: delegate_delete
  project: 灵文1号
  context: "用户刚让内容代理创建了 docs/wechat-smoke.md"
  expected_behavior: "Lead 委派 dev，子代理 delete_file，短回复含「详细」"
  source: "真机脱敏"
```

## 6. 收集规模建议

| 层级 | 条数 | 说明 |
|------|------|------|
| 底线 | ≥50 | 与当前自动化目录同量级 |
| 推荐 | 80–120 | 每意图 3–5 种说法 |
| 理想 | 200+ | 含负样本、失败链、多轮续问 |

当前仓库已自动化 **55 条目录 + 48 条数据驱动 pytest**（见 `utterance_catalog.yaml`），你们补充的应优先覆盖：

- 真机日志里**出现但目录没有**的说法
- **同一意图的口语变体**（比全新场景更重要）
- 多轮上下文（`context` 字段写清楚）
